# CodeMCP 项目规划总结

## 项目概述
CodeMCP 是一个基于 MCP（Model Context Protocol）协议的 AI 协同编排与执行服务器，支持多个 agent 协同执行软件开发任务。系统通过四个核心角色构建了一个具备"自我修复"能力的 AI 生产力闭环。

## 已完成的设计工作

### 1. 架构设计
- 定义了四层核心架构：Planner、Gateway、Console、Executor
- 设计了基于 FastAPI 的 RESTful API 网关
- 规划了使用 Typer + prompt-toolkit 的交互式 Console
- 制定了 MCP 协议接口规范

### 2. 数据模型设计
- 四层数据模型：System → Block → Feature → Test
- 完整的 SQLAlchemy ORM 实现
- 状态流转规则和级联更新逻辑
- 数据库迁移策略（Alembic）

### 3. 核心功能设计
- **状态机系统**: 管理任务生命周期和状态流转
- **窗口化执行机制**: 深度为5的执行窗口，支持优先级调度
- **失败处理机制**: 自动重试、级联中止、重新规划
- **实时监控系统**: WebSocket 事件推送和实时日志

### 4. 接口规范
- **Gateway REST API**: 完整的 RESTful API 接口
- **MCP 协议接口**: Planner 和 Executor 专用接口
- **Console CLI 接口**: 交互式命令行工具
- **WebSocket 事件接口**: 实时状态更新

### 5. 项目结构
- 完整的 Python 包结构设计
- 模块化组件分离：models、api、core、cli、mcp
- 配置管理和环境变量支持
- 测试框架和持续集成配置

### 6. 部署方案
- Docker 容器化部署
- Kubernetes 集群部署
- 监控和运维方案（Prometheus + Grafana）
- 备份和恢复策略

## 技术栈选择

### 后端技术
- **语言**: Python 3.9+
- **Web框架**: FastAPI
- **ORM**: SQLAlchemy 2.0
- **数据库**: SQLite（开发）/ PostgreSQL（生产）
- **异步框架**: asyncio
- **消息队列**: 内置窗口化队列

### 前端/CLI 技术
- **CLI框架**: Typer（基于 Click）
- **交互式界面**: prompt-toolkit
- **终端美化**: Rich
- **配置管理**: pydantic-settings

### 开发工具
- **测试框架**: pytest + pytest-asyncio
- **代码格式化**: Black + isort
- **代码检查**: ruff + mypy
- **文档生成**: MkDocs + mkdocstrings

## 核心特性实现

### 1. 自我修复能力
- 当测试失败时自动重试（最多3次）
- 重试次数用尽后级联中止所属模块
- Planner 可重新规划更细粒度的任务
- 结合 Git 的确定性回滚机制

### 2. 人工接管（HITL）
- Console 提供完整的队列管理功能
- 支持实时暂停、恢复、清理操作
- 允许手动修正任务状态和优先级
- 全链路日志监测和审计

### 3. 确定性交付
- 任务交付以"硬核测试"为准
- 每个任务执行前记录 Git 状态
- 失败时支持回滚到安全点
- 完整的执行日志和测试结果记录

### 4. 逻辑隔离
- 支持多套 System 独立运行
- 防止不同业务领域的记忆干扰
- 独立的配置和资源管理
- 可扩展的插件架构

## 风险评估和缓解措施

### 技术风险
1. **异步编程复杂性**
   - 缓解：使用成熟的 asyncio 框架，编写详细的文档和示例

2. **数据库并发问题**
   - 缓解：使用 SQLAlchemy 的连接池，实现乐观锁机制

3. **窗口调度算法效率**
   - 缓解：实现可配置的调度策略，支持算法替换

### 业务风险
1. **Planner 与 Gateway 的集成复杂度**
   - 缓解：提供详细的 MCP 协议文档和客户端库

2. **大规模任务处理的性能**
   - 缓解：实现任务分片和批量处理，支持水平扩展

3. **系统可用性和可靠性**
   - 缓解：实现健康检查、自动故障转移、监控告警
